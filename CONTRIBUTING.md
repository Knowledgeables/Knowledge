# Contributing

## Branch Naming

<type>(<scope>)-<keywords>

- feat/<scope>-short-description
- fix/<scope>-short-description
- chore/<scope>-short-description
- docs/<scope>-short-description
- refactor/<scope>-short-description

* lowercase only
* use - not _
* max 50 chars

Example:
feat/user-registration

---

## Commit Convention

We follow Conventional Commits:

<type>(<scope>): <short summary>

- feat(scope): new feature
- fix(scope): bug fix
- chore(scope): maintenance
- docs: documentation
- refactor(scope): code refactor
- test(scope): tests

Example:
feat(user): implement registration

---

## Issues

- Every change must be linked to an issue
- Use correct issue type (feature, bug, chore, docs, epic)
- Add domain label (database, user, docs, etc.)
- Acceptance criteria required

---

## Pull Requests
- No direct pushes to main
- Link related issue (Closes #12)
- Keep PRs small and focused
